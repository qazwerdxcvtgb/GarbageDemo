# 基础系统

<a id="基础系统概述"></a>
## 系统概述

基础系统包含项目的核心管理类和玩家控制相关的基础组件，为其他系统提供输入处理、事件管理和交互机制支持。

**包含的核心类**：
- **GameEvents**：游戏通用事件定义，存放所有通用事件类型
- **GameManager**：游戏管理器，使用单例模式管理全局事件和状态
- **InputSystem**：输入系统，Unity自动生成的输入动作映射
- **PlayerMove**：玩家移动控制器，处理玩家移动和输入
- **InteractionPoint**：交互点基类，用于创建可交互物体
- **CharacterState**：角色状态管理器，管理玩家体力值、金币等状态数据

### 1. GameEvents（游戏通用事件定义）

**文件路径**：`Assets/Script/GameEvents.cs`

**类型**：静态事件类定义文件

**功能说明**：
- 存放项目中所有通用的事件类型定义
- 避免事件类型分散在各个脚本中
- 提供统一的事件类型管理

**包含的事件类型**：

#### IntValueChangedEvent
```csharp
[System.Serializable]
public class IntValueChangedEvent : UnityEvent<int> { }
```

**用途**：
- 用于传递整数类型参数的事件
- 适用于：体力值变化、金币变化、疯狂值变化等

**使用示例**：
```csharp
// 定义事件
public IntValueChangedEvent OnHealthChanged;

// 初始化事件
OnHealthChanged = new IntValueChangedEvent();

// 订阅事件
OnHealthChanged.AddListener(OnHealthChangedHandler);

// 触发事件
OnHealthChanged?.Invoke(newHealthValue);

// 事件处理方法
void OnHealthChangedHandler(int newHealth)
{
    Debug.Log($"体力变化为: {newHealth}");
}
```

**设计理念**：
- ✅ **职责单一**：专门存放通用事件类型
- ✅ **便于扩展**：后续可添加更多通用事件类型
- ✅ **避免重复**：多个脚本共享同一事件定义
- ✅ **提升维护性**：集中管理，便于查找和修改

**未来扩展示例**：
```csharp
// 可以添加更多通用事件类型
[System.Serializable]
public class FloatValueChangedEvent : UnityEvent<float> { }

[System.Serializable]
public class StringValueChangedEvent : UnityEvent<string> { }

[System.Serializable]
public class BoolValueChangedEvent : UnityEvent<bool> { }
```

### 5. InteractionPoint（交互点基类）

**文件路径**：`Assets/Script/InteractionPoint.cs`

**类型**：MonoBehaviour（基类）

**功能说明**：
- 用于创建可与玩家交互的物体（如NPC、宝箱、道具等）
- 检测玩家进入/离开交互范围
- 控制交互提示动画的显示/隐藏
- 使用`protected`修饰符方便子类继承和扩展

**主要成员**：
```csharp
// 动画控制器（用于显示/隐藏提示）
protected Animator animator;

// 玩家进入触发范围
protected virtual void OnTriggerEnter2D(Collider2D other)

// 玩家离开触发范围
protected virtual void OnTriggerExit2D(Collider2D other)
```

**使用要求**：
1. 游戏对象必须有`Collider2D`组件，并勾选"Is Trigger"
2. 游戏对象必须有`Animator`组件
3. Animator中需要有`isNear`布尔参数
4. 玩家对象必须有"Player"标签

**扩展示例**：
```csharp
// 继承InteractionPoint创建具体的交互物体
public class TreasureChest : InteractionPoint
{
    protected override void OnTriggerEnter2D(Collider2D other)
    {
        base.OnTriggerEnter2D(other); // 调用基类显示提示
        
        // 添加自定义逻辑
        // 例如：订阅交互事件
        GameManager.Instance.OnInteractionPressed.AddListener(OpenChest);
    }
    
    protected override void OnTriggerExit2D(Collider2D other)
    {
        base.OnTriggerExit2D(other); // 调用基类隐藏提示
        
        // 取消订阅
        GameManager.Instance.OnInteractionPressed.RemoveListener(OpenChest);
    }
    
    private void OpenChest()
    {
        Debug.Log("宝箱已打开！");
        // 打开宝箱的逻辑
    }
}
```

**工作流程**：
```
1. 玩家靠近交互点
   → OnTriggerEnter2D触发
   → 设置animator.SetBool("isNear", true)
   → 显示交互提示图标

2. 玩家离开交互点
   → OnTriggerExit2D触发
   → 设置animator.SetBool("isNear", false)
   → 隐藏交互提示图标

3. 玩家在范围内按F键
   → PlayerMove.OnInteraction触发
   → 调用GameManager.TriggerInteractionPressed()
   → 订阅者响应交互事件
```

# CardSystem - 卡牌系统

## 系统概述

### 功能介绍
卡牌系统是一个基于Unity的可扩展卡牌管理框架，支持：
- **卡片数据管理**：使用ScriptableObject可视化配置卡片属性
- **效果系统**：支持揭示、捕获、食用三个阶段的效果触发
- **卡池管理**：提供筛选、抽卡、查询等功能
- **概率控制**：支持基于权重的抽卡算法
- **外部扩展**：预留CSV加载接口，支持未来Mod扩展

### 使用场景
- 🎣 **钓鱼系统**：玩家钓鱼时抽取卡片，触发揭示和捕获效果
- 🎒 **背包系统**：玩家在背包中使用卡片，触发食用效果
- 🏪 **商店系统**：卡片可以按价值售卖

### 技术选型
- **ScriptableObject**：适合Unity的数据存储方式，方便Inspector编辑
- **单例模式**：CardPool使用单例确保全局唯一访问点
- **继承体系**：效果系统采用抽象基类，易于扩展
- **LINQ查询**：高效的筛选和查询功能

#### Effect_AddRandomHealth（随机增加体力值）

**文件路径**：`Assets/Script/CardSystem/Effects/Effect_AddRandomHealth.cs`

**功能说明**：
- 在最小值和最大值之间随机增加体力
- 包含最小值和最大值（闭区间）
- 自动查找场景中的player对象
- 参数可在Inspector中配置，带自动验证
- 执行时输出随机到的具体数值

**参数配置**：
- `minHealthAmount` (int)：最小体力增加值，≥0（默认值：1）
- `maxHealthAmount` (int)：最大体力增加值，≥最小值（默认值：10）

**使用场景**：
- 增加卡牌效果的随机性和趣味性
- 风险与收益的平衡设计
- 不确定性的游戏体验

**代码实现**：
```csharp
/// <summary>
/// 随机增加体力值效果
/// 创建日期：2026-01-21
/// </summary>

using UnityEngine;

namespace CardSystem
{
    /// <summary>
    /// 在最小值和最大值之间随机增加体力
    /// </summary>
    [CreateAssetMenu(fileName = "Effect_AddRandomHealth", menuName = "CardSystem/Effects/AddRandomHealth", order = 12)]
    public class Effect_AddRandomHealth : CardEffect
    {
        [Header("随机体力增加设置")]
        [Tooltip("最小体力增加值（必须≥0）")]
        [SerializeField] private int minHealthAmount = 1;

        [Tooltip("最大体力增加值（必须≥最小值）")]
        [SerializeField] private int maxHealthAmount = 10;

        /// <summary>
        /// 最小体力增加量属性（带参数验证）
        /// </summary>
        public int MinHealthAmount
        {
            get => minHealthAmount;
            set
            {
                minHealthAmount = Mathf.Max(0, value); // 确保≥0
                // 确保最大值不小于最小值
                if (maxHealthAmount < minHealthAmount)
                    maxHealthAmount = minHealthAmount;
            }
        }

        /// <summary>
        /// 最大体力增加量属性（带参数验证）
        /// </summary>
        public int MaxHealthAmount
        {
            get => maxHealthAmount;
            set
            {
                maxHealthAmount = Mathf.Max(minHealthAmount, value); // 确保≥最小值
            }
        }

        /// <summary>
        /// 执行效果：随机增加玩家体力
        /// </summary>
        public override void Execute()
        {
            // 查找玩家对象
            GameObject player = GameObject.Find("player");
            if (player == null)
            {
                Debug.LogWarning("[Effect_AddRandomHealth] 未找到玩家对象（名称应为'player'）");
                return;
            }

            // 获取角色状态组件
            CharacterState playerState = player.GetComponent<CharacterState>();
            if (playerState == null)
            {
                Debug.LogError("[Effect_AddRandomHealth] 玩家对象缺少CharacterState组件");
                return;
            }

            // 生成随机体力值（闭区间，包含最小值和最大值）
            int randomHealth = Random.Range(minHealthAmount, maxHealthAmount + 1);

            // 增加体力
            playerState.ModifyHealth(randomHealth);

            // 输出日志（包含随机值和范围）
            Debug.Log($"[Effect_AddRandomHealth] 执行效果：随机增加体力 +{randomHealth} (范围:{minHealthAmount}-{maxHealthAmount})，当前体力：{playerState.CurrentHealth}/{playerState.MaxHealth}");
        }
    }
}

### 如何添加新的效果类型

参考已实现的`Effect_AddHealth`、`Effect_ModifySanity`、`Effect_AddRandomHealth`三个效果类，您可以按照以下步骤创建新的效果类型。

**步骤1**：创建新的效果脚本

**示例**：创建一个增加金币的效果

```csharp
using UnityEngine;

namespace CardSystem
{
    /// <summary>
    /// 增加玩家金币效果
    /// </summary>
    [CreateAssetMenu(fileName = "Effect_AddGold", menuName = "CardSystem/Effects/AddGold", order = 13)]
    public class Effect_AddGold : CardEffect
    {
        [Header("金币增加设置")]
        [Tooltip("增加的金币数量")]
        [SerializeField] private int goldAmount = 10;
        
        public int GoldAmount
        {
            get => goldAmount;
            set => goldAmount = Mathf.Max(0, value); // 参数验证
        }
        
        public override void Execute()
        {
            // 查找玩家对象
            GameObject player = GameObject.Find("player");
            if (player == null)
            {
                Debug.LogWarning("[Effect_AddGold] 未找到玩家对象");
                return;
            }
            
            // 获取角色状态组件
            CharacterState playerState = player.GetComponent<CharacterState>();
            if (playerState == null)
            {
                Debug.LogError("[Effect_AddGold] 玩家对象缺少CharacterState组件");
                return;
            }
            
            // 增加金币
            playerState.ModifyGold(goldAmount);
            
            // 输出日志
            Debug.Log($"[Effect_AddGold] 执行效果：增加金币 +{goldAmount}，当前金币：{playerState.GoldAmount}");
        }
    }
}
```

### 如何实现CSV加载（未来Mod支持）

**CSV格式设计**：
```csv
名称,深度,体积,价值,消耗体力,类型,权重,揭示效果,捕获效果,食用效果
小鱼,Depth1,Small,50,3,Pure,1.0,Print1,Print2|Print3,
中鱼,Depth2,Medium,150,8,Pure,0.8,Print2,Print3,Print4
大鱼,Depth3,Large,500,20,Corrupted,0.3,Print5,Print5,Print5
```

**实现思路**：
1. 在 `CardExternalLoader.LoadCardsFromCSV()` 中：
   - 使用 `System.IO.File.ReadAllLines()` 读取CSV
   - 跳过表头，遍历每一行
   - 调用 `CSVLineToCardData()` 转换数据

2. 在 `CSVLineToCardData()` 中：
   - 使用 `string.Split(',')` 分割列
   - 使用 `ScriptableObject.CreateInstance<CardData>()` 创建实例
   - 解析枚举：`Enum.Parse<CardDepth>(fields[1])`
   - 解析效果列表：按 `|` 分割，从Resources加载对应效果资产

3. 在 `CardPool` 中添加动态加载方法：
```csharp
public void LoadCardsFromExternalFile(string csvPath)
{
    List<CardData> externalCards = CardExternalLoader.LoadCardsFromCSV(csvPath);
    allCards.AddRange(externalCards);
    Debug.Log($"从外部加载了 {externalCards.Count} 张卡片");
}
```

    └── ...

### 2. 权重配置建议

- **等概率**：所有卡片权重设为 `1.0`
- **稀有卡**：深度3的卡片权重可设为 `0.3`（更稀有）
- **常见卡**：深度1的卡片权重可设为 `2.0`（更常见）

**权重计算示例**：
```
卡片A：权重 2.0
卡片B：权重 1.0
卡片C：权重 0.5
总权重：3.5

卡片A概率：2.0 / 3.5 ≈ 57%
卡片B概率：1.0 / 3.5 ≈ 29%
卡片C概率：0.5 / 3.5 ≈ 14%
```

### 3. 效果数量限制

- 系统支持每个阶段 **0-5个效果**
- 没有硬性代码限制，但建议不超过5个（性能和平衡性考虑）
- 效果按列表顺序依次执行

### 4. 性能优化建议

- 卡片数量在200张以内，当前架构性能足够
- 如果卡片数量超过1000张，建议：
  - 使用字典（Dictionary）代替列表存储
  - 添加卡片索引缓存
  - 延迟加载（Lazy Loading）

### 5. 单例生命周期

- `CardPool` 使用 `DontDestroyOnLoad`，场景切换时不会销毁
- 如果需要重置卡池，调用 `ReloadAllCards()`

### 6. 空引用检查

在使用抽卡结果前，务必检查是否为 `null`：
```csharp
CardData card = CardPool.Instance.DrawRandomCard();

    // 使用卡片
}
else
{
    Debug.LogWarning("卡池为空或抽卡失败");
}

- **体力显示UI**：显示玩家当前体力值和最大体力值
- **金币显示UI**：显示玩家金币数量，带帧动画和数字渐变效果
- **疯狂值显示UI**：显示世界疯狂值，根据疯狂等级改变文字底色（2026-01-21新增）
- **卡片选择面板**：显示3张卡牌供玩家选择
- **卡片按钮**：显示卡牌详细信息并处理点击
- **单选机制**：确保只能选择一张卡牌
- **视觉反馈**：选中状态高亮显示
- **手牌UI系统**：显示和管理玩家手牌列表，支持选中和食用卡牌（2026-01-21新增）

### 使用场景
- 💚 **体力显示**：实时显示玩家体力状态
- 💰 **金币显示**：实时显示金币数量，金币变化时播放动画
- 🌀 **疯狂值显示**：实时显示世界疯狂值，颜色从黄到红渐变
- 🎣 **钓鱼系统**：钓鱼后选择要捕获的卡牌
- 🎁 **奖励系统**：从多张奖励卡牌中选择一张
- 🎲 **随机事件**：触发随机事件时选择卡牌
- 🎒 **手牌管理**：查看和使用手牌中的卡牌，食用卡牌触发效果

3. 在Inspector中分配healthSlider和healthText引用

**使用方式**：

#### 在Unity中配置

```
1. 在Canvas下创建UI容器GameObject（如"HealthDisplay"）
   - 定位到左上角
   
2. 在容器下创建Slider和TextMeshProUGUI组件

3. 给容器添加HealthDisplayUI脚本

   - 将TextMeshProUGUI拖到healthText字段

**工作流程**：
```
1. Start时查找player对象

   ↓
3. 订阅OnHealthChanged和OnMaxHealthChanged事件
   ↓
4. 初始化UI显示
   ↓
5. 体力变化时自动触发UpdateHealthUI
   ↓
6. 更新Slider值（比例）和Text显示（数值）

**数据更新**：
- **Slider.value**：`currentHealth / maxHealth`（0-1之间的浮点数）
- **Text.text**：`"{currentHealth}/{maxHealth}"`（如"10/15"）

**注意事项**：

### 2. GoldDisplayUI（金币显示UI）

**文件路径**：`Assets/Script/UI/GoldDisplayUI.cs`

**类型**：MonoBehaviour

**主要功能**：
- 自动查找玩家CharacterState组件
- 订阅金币变化事件
- 实时更新金币数量显示
- 控制金币图标的帧动画播放
- 实现数字渐变增加效果（智能步长优化）

**主要成员**：
```csharp
// TextMeshProUGUI组件（显示金币数量）
public TextMeshProUGUI goldText

// Animator组件（金币图标动画控制）
public Animator coinAnimator

// 金币动画Bool参数名
public string coinAnimParamName = "isPlayingCoinAnim"

// 变化频率（次/秒）
public float changeFrequency = 10f

// 玩家状态组件引用
private CharacterState playerState

// 当前显示的金币数量
private int displayedGold

// 金币变化协程引用
private Coroutine goldChangeCoroutine
```

**使用要求**：
1. 场景中必须有名为"player"的GameObject
2. player对象上必须有CharacterState组件
3. 在Inspector中分配goldText和coinAnimator引用
4. 在Animator Controller中创建名为"isPlayingCoinAnim"的Bool参数，并正确配置状态机

**使用方式**：

#### 在Unity中配置

**步骤1：创建帧动画**

1. **准备Sprite图片**：
   - 将金币帧图片序列导入Unity
   - 设置Texture Type为"Sprite (2D and UI)"

2. **创建Animation Clip**：
   - 在Project窗口：Create → Animation
   - 创建两个动画：
     - "CoinIdle"（可选，静止状态，可以是单帧或空动画）
     - "CoinPlaying"（金币旋转动画，包含所有帧）

3. **编辑CoinPlaying动画**：
   - 打开Animation窗口（Window → Animation → Animation）
   - 创建临时Image对象预览
   - 添加属性：Image → Source Image
   - 批量拖拽所有帧图片到时间轴（推荐）
   - 或手动逐帧添加关键帧
   - 调整帧率（建议10-20 FPS）
   - **重要**：勾选Loop循环播放

4. **创建Animator Controller**：
   - Create → Animator Controller
   - 命名为"CoinAnimator"

5. **配置Animator状态机**（重要）：
   - 打开Animator窗口
   - 创建Bool参数："isPlayingCoinAnim"
   - 创建两个状态：
     - **Idle状态**：拖入CoinIdle动画（或创建空状态）
     - **Playing状态**：拖入CoinPlaying动画
   - 设置Idle为默认状态（橙色）
   - 创建过渡：
     - **Idle → Playing**：
       - 条件：isPlayingCoinAnim = true
       - Has Exit Time：取消勾选
       - Transition Duration：0（无过渡）
     - **Playing → Idle**：
       - 条件：isPlayingCoinAnim = false
       - Has Exit Time：取消勾选
       - Transition Duration：0（无过渡）

**步骤2：配置UI对象**

```
1. 在Canvas下创建UI容器GameObject："GoldDisplay"
   - 定位到左上角体力UI下方
   
2. 在GoldDisplay下创建金币图标：
   - 创建UI → Image，命名为"CoinIcon"
   - 添加Animator组件，指定Controller为CoinAnimator
   - 设置Source Image为金币第一帧
   
3. 在GoldDisplay下创建金币文本：
   - 创建UI → Text - TextMeshPro，命名为"GoldText"
   
4. 给GoldDisplay添加GoldDisplayUI脚本
   
5. 在Inspector中拖拽引用：
   - goldText ← GoldText组件
   - coinAnimator ← CoinIcon的Animator组件
   
6. 配置参数：
   - Coin Anim Param Name = "isPlayingCoinAnim"
   - Change Frequency = 10（每秒10次，间隔0.1秒）
```

**工作流程**：
```
1. Start时查找player对象
   ↓
2. 获取CharacterState组件
   ↓
3. 订阅OnGoldChanged事件
   ↓
4. 初始化显示当前金币数量
   ↓
5. 金币变化时触发UpdateGoldUI
   ↓
6. 设置Animator Bool参数为true，开始播放金币动画
   ↓
7. 启动数字渐变协程
   ↓
8. 根据变化量智能选择步长
   ↓
9. 逐步更新显示的金币数字
   ↓
10. 数字渐变完成，设置Animator Bool参数为false，停止动画
```

**智能步长优化**：
- **变化量 ≤10**：每次+1（如+7 = 7步）
- **变化量 10-100**：每次+10（如+50 = 5步）
- **变化量 100-1000**：每次+100（如+500 = 5步）
- **变化量 >1000**：每次+1000（如+5000 = 5步）

**数据更新**：
- **变化间隔**：`1 / changeFrequency`（如频率10 = 间隔0.1秒）
- **Text.text**：直接显示金币数量（如"123"）
- **动画控制**：
  - 金币开始变化时：`animator.SetBool("isPlayingCoinAnim", true)` → 开始循环播放
  - 金币停止变化时：`animator.SetBool("isPlayingCoinAnim", false)` → 停止播放

**参数配置建议**：
- **changeFrequency = 10**：每秒10次，间隔0.1秒（推荐，平衡速度）
- **changeFrequency = 20**：每秒20次，间隔0.05秒（快速变化）
- **changeFrequency = 5**：每秒5次，间隔0.2秒（慢速变化）

**调试功能**：
脚本提供了Context Menu调试方法，在Inspector中右键组件可测试：
- "测试增加10金币"
- "测试增加500金币"
- "测试减少5金币"

**注意事项**：
- 脚本会在OnDestroy时自动取消事件订阅、停止协程和停止动画
- 如果玩家对象名称不是"player"，需要修改代码中的查找逻辑
- **重要**：Animator必须包含名为"isPlayingCoinAnim"的Bool参数（不是Trigger）
- **重要**：CoinPlaying动画必须勾选Loop循环播放
- **重要**：状态过渡必须取消Has Exit Time，否则动画停止会有延迟
- 金币数量在CharacterState中已限制≥0
- 如果金币快速连续变化，旧协程会被停止并启动新协程
- 动画会在数字渐变完成时自动停止，无需手动控制

**帧动画创建详细步骤**：

1. **批量拖拽方式（最快）**：
   - 选中所有金币帧图片（coin_01 ~ coin_08）
   - 直接拖到Animation窗口时间轴
   - Unity自动创建关键帧序列

2. **手动添加方式**：
   - 在时间轴0:00位置，设置第一帧
   - 移动到0:05（假设20帧/秒），设置第二帧
   - 重复直到所有帧添加完成

3. **帧率调整**：
   - 在Animation窗口右上角设置Samples（帧率）
   - 建议10-20 FPS适合金币动画
   - 帧率越高，动画越流畅但文件越大

4. **Animator配置要点**：
   - 使用Bool参数而非Trigger参数
   - CoinPlaying动画必须勾选Loop循环播放
   - 状态过渡必须取消Has Exit Time
   - Transition Duration设为0，实现即时切换
   - 这样动画会在数字变化期间持续播放，数字停止时立即停止

### 4. CardSelectionPanel（卡片选择面板）

**文件路径**：`Assets/Script/UI/CardSelectionPanel.cs`

**类型**：MonoBehaviour

**更新日期**：2026-01-26

**主要功能**：
- 管理9个CardButton（3行x3列）
- 根据深度参数抽取多行卡牌（深度1=1行，深度2=2行，深度3=3行）
- 处理单选逻辑
- 实现两阶段显示流程（选择阶段 → 揭示阶段）
- 控制确认/捕获/放弃按钮状态
- 体力检查与扣除
- 执行揭示、捕获效果并加入手牌
- 支持空牌占位（卡池不足时显示空白卡）
- 使用ScrollView支持多行滚动显示

**主要成员**：
```csharp
// UI组件
public CardButton[] cardButtons        // 9个卡片按钮（3行x3列）
public GameObject[] cardRows           // 3行容器对象（Row1、Row2、Row3）
public TextMeshProUGUI[] depthLabels   // 3个深度标签（深度1、深度2、深度3）
public Button confirmButton            // 确认按钮
public Button captureButton            // 捕获按钮
public Button abandonButton            // 放弃按钮
public GameObject panelRoot            // 面板根对象（PanelContent，用于显隐控制）

// 公开方法
public void OpenPanel(CardDepth drawCardDepth)  // 打开面板（根据深度抽取多行）
public void ClosePanel()                        // 关闭面板
public void OnCardButtonClicked(CardButton clickedButton)  // 卡片按钮点击回调
```

**⚠️ panelRoot 重要说明**：
- `panelRoot` 字段必须指向 `PanelContent` 子对象，**不能指向脚本对象本身**
- `CardSelectionPanel` 脚本挂载对象必须始终保持 active
- `PanelContent`（panelRoot）用于控制显隐，初始状态为 inactive
- 通过 `panelRoot.SetActive(true/false)` 来显示/隐藏面板
- 如果 `panelRoot` 指向脚本对象本身，隐藏时会导致脚本被禁用，无法再次打开
- 代码中有检查逻辑：如果检测到 `panelRoot == this.gameObject`，会输出警告

**层级结构要求**：

正确的层级结构：
```
CardSelectionPanel（脚本挂载，勾选✓）
└── PanelContent（panelRoot指向这里，不勾选✗）
    ├── ScrollView
    │   └── Content
    │       ├── Row1 (DepthLabel1 + CardButton[0-2])
    │       ├── Row2 (DepthLabel2 + CardButton[3-5])
    │       └── Row3 (DepthLabel3 + CardButton[6-8])
    ├── ConfirmButton
    ├── CaptureButton
    └── AbandonButton
```

**Inspector 配置要点**：
1. `Panel Root` 字段 → 拖入 `PanelContent` 对象
2. `Card Buttons` 数组 (Size=9) → 按顺序拖入9个CardButton
3. `Card Rows` 数组 (Size=3) → 拖入 Row1、Row2、Row3
4. `Depth Labels` 数组 (Size=3) → 拖入3个深度标签
5. 按钮字段 → 拖入对应的按钮

**两阶段流程**：

#### 第一阶段：选择阶段
- 显示3张卡牌的体积信息（小/中/大）
- 玩家可选择一张卡牌
- 确认按钮可见，捕获/放弃按钮隐藏
- 选中卡牌后点击确认按钮进入第二阶段

#### 第二阶段：揭示阶段
- 触发揭示效果（`TriggerRevealEffects()`）
- 显示选中卡牌的所有详细信息
- 确认按钮隐藏，捕获/放弃按钮显示
- **禁用其他卡片按钮**（防止玩家切换选择，保持选中按钮可用但点击无响应）
- **阻止卡牌点击响应**：在此阶段点击任何卡牌按钮都不会触发选择逻辑
- 根据玩家体力检查捕获按钮是否可用
  - 体力足够：按钮可点击
  - 体力不足：按钮置灰
- 点击捕获：扣除体力 → 触发捕获效果 → 添加到手牌 → 关闭UI
- 点击放弃：直接关闭UI

**体力检查机制**：
- 在揭示阶段自动检查玩家体力
- 如果体力 < 卡牌消耗，捕获按钮置灰
- 点击捕获时再次检查体力（双重保险）
- 扣除体力后显示日志

**使用要求**：
1. 场景中必须有名为 `"player"` 的GameObject
2. player对象必须挂载 `CharacterState` 组件
3. 在Inspector中配置所有UI组件引用

### 技术要点

#### DOTween动画

使用的DOTween API：
- `RectTransform.DOAnchorPosY(float endValue, float duration)` - Y轴位置动画
- `CanvasGroup.DOFade(float endValue, float duration)` - 透明度动画（未使用，背景保持透明）
- `.SetEase(Ease easeType)` - 设置缓动类型（OutCubic/InCubic）
- `.OnComplete(TweenCallback callback)` - 动画完成回调
- `DOTween.Kill(object targetOrId)` - 停止动画

#### ScrollView配置

- **Horizontal Layout Group**：自动横向排列卡片，间距20像素
- **Content Size Fitter**：自动调整Content大小以适应所有卡片
- **Mask组件**：遮罩超出Viewport的内容，确保手牌不会超出显示范围

#### 背景遮罩

- **完全透明**：Alpha=0，视觉上不可见
- **检测点击**：Button组件响应点击事件，用于关闭面板
- **全屏覆盖**：确保点击面板外任何位置都能触发关闭

### 注意事项

1. **DOTween依赖**：确保项目已安装DOTween插件
2. **命名空间**：需要`using DG.Tweening;`
3. **事件订阅管理**：务必在OnDestroy中取消订阅，避免内存泄漏
4. **动画状态**：使用`isAnimating`标志避免动画播放期间的重复操作
5. **背景遮罩透明**：BackgroundBlocker的Image Alpha设为0，完全透明
6. **锚点设置**：HandButton和HandListPanel必须使用Bottom-Left锚点
7. **卡片生成**：每次打开面板都重新生成卡片，关闭时销毁（可后续优化为对象池）
8. **位置参数可调整**：所有位置参数（handButtonInitialY、handButtonTargetY等）都可以在Inspector中调整，无需修改代码。调整面板大小时，记得同步调整这些参数。
9. **⚠️ 脚本执行顺序**：HandUIPanel在`Start`中订阅事件（而非`Awake`），避免HandManager.Instance为null的问题

### 调整面板大小示例

如果您想调整面板高度为600像素：
1. 修改HandListPanel的Height为600
2. 在Inspector中修改HandUIPanel脚本的参数：
   - `Panel Height` = 600
   - `Hand Button Target Y` = 600
   - `Hand List Panel Initial Y` = -600
   - `Hand List Panel Target Y` = 0（保持不变）

# FishShopSystem - 鱼店系统

<a id="鱼店系统概述"></a>
## 系统概述

**创建日期**：2026-01-21

### 功能介绍
鱼店系统允许玩家将手牌中的鱼类卡牌售卖给鱼店，获得金币。售价根据世界疯狂等级和鱼类类型（纯净/污秽）动态调整。售卖污秽鱼会增加世界疯狂值。

**核心功能**：
- 🏪 **交互点**：玩家与鱼店NPC交互打开鱼店面板
- 🎣 **鱼类售卖**：从手牌拖拽鱼类卡牌到鱼店面板（卡牌从手牌移除）
- ↩️ **拖回手牌**：从鱼店拖拽卡牌回到手牌（卡牌从鱼店移除）
- 💰 **动态售价**：根据世界疯狂等级计算售价（6个等级）
- 🌀 **疯狂值影响**：售卖污秽鱼增加世界疯狂值
- 🔄 **可取消**：关闭面板时鱼店中的所有卡牌自动返还到手牌

### 使用场景
- 🎮 **卡牌售卖**：玩家将多余的鱼类卡牌换成金币
- 📈 **策略选择**：根据疯狂等级选择售卖时机（纯净鱼vs污秽鱼）
- ⚖️ **风险收益**：售卖污秽鱼获得更高收益但增加疯狂值

### 技术选型
- **拖拽交互**：使用Unity EventSystem接口（IBeginDragHandler等）
- **单例模式**：FishShopPanel使用懒加载单例
- **动态计算**：FishPriceCalculator静态工具类
- **UI组件**：ScrollView横向布局，类似手牌UI

## 注意事项

### 1. 拖拽限制

- **只能拖拽鱼类**：非鱼类卡牌（装备、杂鱼等）无法拖拽到鱼店
- **只在鱼店打开时**：鱼店面板未打开时，拖拽功能不生效
- **手牌必须打开**：鱼店面板打开时会自动打开手牌面板

### 2. 单例生命周期

- **FishShopPanel使用懒加载单例**：无需担心初始化顺序
- **场景中必须有FishShopPanel对象**：可以初始inactive，但必须存在
- **Instance.get会自动查找**：首次访问时通过FindObjectOfType查找

### 3. UI层级

- **拖拽预览在最上层**：dragPreview.transform.SetAsLastSibling()
- **背景遮罩在最底层**：BackgroundBlocker在Hierarchy最上面

### 4. 数据同步

- **卡牌数据来自HandManager**：鱼店中的卡牌是HandManager中卡牌的引用
- **售卖时才真正移除**：添加到鱼店不会立即从HandManager移除
- **关闭时无需返还**：关闭面板只清空fishInShop列表，卡牌本来就在HandManager中

### 5. 疯狂值影响

- **只有污秽鱼增加疯狂值**：纯净鱼不影响疯狂值
- **疯狂值增加量 = 污秽鱼数量**：售卖3条污秽鱼 = 疯狂值+3
- **疯狂值影响售价**：疯狂值变化会改变未来的售价

# 钓鱼抽牌多行深度功能

**功能名称**：钓鱼抽牌多行深度系统

**创建日期**：2026-01-26

**功能概述**：
根据钓鱼地点的深度参数，从不同深度的卡池抽取卡牌并分行显示。支持空牌占位，当某深度卡池不足3张时显示空白卡。

---

## 功能说明

### 抽牌规则

**深度1**：
- 第1行：从深度1卡池抽取3张卡牌
- 总计：3张卡牌

**深度2**：
- 第1行：从深度1卡池抽取3张卡牌
- 第2行：从深度2卡池抽取3张卡牌
- 总计：6张卡牌

**深度3**：
- 第1行：从深度1卡池抽取3张卡牌
- 第2行：从深度2卡池抽取3张卡牌
- 第3行：从深度3卡池抽取3张卡牌
- 总计：9张卡牌

### 空牌机制

**触发条件**：
- 当某深度卡池不足3张时，用空牌占位

**空牌特性**：
- 显示文本："空白"
- 背景半透明（alpha = 0.5）
- 不可选中（button.interactable = false）
- 不可揭示
- 不可捕获
- 点击无响应

**空牌统计**：
- 在面板打开时统计（用户看到空牌时）
- 记录到 `GameManager.EmptyCardDrawnCount`
- 后期可用于功能扩展（如成就、特殊事件等）

### Unity编辑器操作

**⚠️ 重要前提：确认层级结构**

在开始之前，必须确保以下层级结构：

CardSelectionPanel（脚本挂载，勾选active）

## 注意事项

### 1. 卡池充足性
- 确保每个深度至少有足够的卡牌（建议≥3张）
- 卡池不足时会自动用空牌占位
- 空牌不影响游戏流程，仅作为占位符

### 2. 空牌交互
- 空牌按钮不响应点击
- 空牌不参与选中逻辑
- 空牌不触发任何效果
- 确认按钮只在选中真实卡牌时激活

### 3. ScrollView滚动
- 单行时可能无需滚动（Content高度小于ScrollView高度）
- 多行时自动显示滚动条
- Content Size Fitter 自动调整Content高度

### 4. 深度标签
- 深度标签由脚本自动设置文本
- 深度标签样式可在Inspector中自定义
- 深度标签不参与游戏逻辑，仅用于UI显示

### 5. UI布局调整
- 确认/捕获/放弃按钮固定在面板底部（ScrollView外部，PanelContent内部）
- 深度标签宽度建议固定（如100像素）
- CardButton自动填充剩余空间

### 6. PanelContent 层级说明（重要）
- **必须保留** `PanelContent` 作为显隐控制层
- `CardSelectionPanel` 对象必须始终 active（脚本才能工作）
- `PanelContent` 初始状态为 inactive（隐藏面板）
- 所有UI元素（ScrollView、按钮等）都必须在 `PanelContent` 内部
- `panelRoot` 字段必须指向 `PanelContent`，不能指向脚本对象本身
- 如果 `panelRoot` 指向错误，会导致面板无法正常显示/隐藏

# ItemSystem - 物品系统（重构版）

**创建日期**：2026-01-26

## 系统概述

### 功能介绍
物品系统是从CardSystem重构而来的全新物品管理框架，支持：
- **类型体系**：鱼类、杂鱼、消耗品、装备四大类型
- **继承架构**：ItemData基类 + 子类特化
- **效果系统**：即时效果（InstantEffect）+ 常驻效果（PassiveEffect）
- **物品池管理**：ItemPool统一管理，支持按类型筛选和抽取
- **装备系统**：EquipmentManager管理装备槽位和被动效果

### 与CardSystem的差异

| 对比项 | CardSystem（旧） | ItemSystem（新） |
|--------|-----------------|-----------------|
| 基类 | CardData（统一结构） | ItemData（抽象基类） |
| 类型 | category字段区分 | 继承体系（FishData、TrashData等） |
| 属性 | 所有属性混在一起 | 类型特有属性分离 |
| 效果 | CardEffect | InstantEffect + PassiveEffect |
| 术语 | "食用" | "使用" |
| 装备 | 不支持 | EquipmentData + EquipmentManager |

## 架构设计

### 继承体系

```
ItemData (抽象基类)
├── FishData (鱼类)
├── TrashData (杂鱼)
├── ConsumableData (消耗品)
└── EquipmentData (装备)

EffectData (抽象基类)
├── InstantEffect (即时效果)
│   ├── InstantEffect_AddHealth
│   ├── InstantEffect_ModifySanity
│   └── InstantEffect_AddRandomHealth
└── PassiveEffect (常驻效果)

EffectConfig (效果配置类)
├── effectType (效果类型枚举)
├── numericValue (数值参数)
├── minValue (最小值)
└── maxValue (最大值)
```

- ✅ **装备系统**：1个鱼竿槽 + 1个渔具槽

### 3. 效果配置系统（2026-01-28更新）

**重要变更**：即时效果系统已从资产引用模式改为内嵌配置模式。

#### 3.1 效果配置系统概述

**旧方式（已废弃）**：
- 需要为每个效果创建ScriptableObject资产
- 在物品资产中引用效果资产
- 参数配置在效果资产中

**新方式（当前）**：
- 无需创建效果资产
- 在物品资产中直接配置效果类型和参数
- 参数直接在物品资产的Inspector中配置

#### 3.2 核心组件

**EffectType（效果类型枚举）**
- 文件路径：`Assets/Script/ItemSystem/Effects/EffectType.cs`
- 定义所有可用的效果类型
- 使用 `[InspectorName]` 属性显示中文名称

**EffectConfig（效果配置类）**
- 文件路径：`Assets/Script/ItemSystem/Effects/EffectConfig.cs`
- 序列化类，包含效果类型和参数
- 字段：
  - `effectType`：效果类型（下拉选择）
  - `numericValue`：数值参数（用于固定数值效果）
  - `minValue`：最小值（用于随机效果）
  - `maxValue`：最大值（用于随机效果）
- 方法：
  - `Execute(EffectContext context)`：执行效果
  - `GetEffectDescription()`：获取效果描述

**EffectFactory（效果工厂类）**
- 文件路径：`Assets/Script/ItemSystem/Effects/EffectFactory.cs`
- 负责根据EffectType创建效果实例并执行
- 维护效果类型到效果类的映射字典
- 根据配置创建EffectContext并设置参数

**EffectConfigDrawer（自定义PropertyDrawer）**
- 文件路径：`Assets/Script/ItemSystem/Editor/EffectConfigDrawer.cs`
- 实现Inspector中的自定义显示
- 功能：
  - 下拉菜单选择效果类型
  - 根据效果类型动态显示对应参数
  - 参数验证提示

#### 3.3 使用方式

**在Unity Editor中配置效果**：

1. 选择或创建物品资产（FishData、TrashData、ConsumableData）
2. 在Inspector中展开"效果配置"区域
3. 在效果列表（revealEffects、captureEffects、useEffects）中添加元素
4. 点击列表项展开，会看到：
   - **效果类型**：下拉菜单（增加体力/修改疯狂值/随机增加体力）
   - **参数配置**：根据选择的类型显示对应参数
     - 增加体力：显示"增加体力值"
     - 修改疯狂值：显示"疯狂值变化量"
     - 随机增加体力：显示"最小体力值"和"最大体力值"（带验证提示）
5. 配置完成后，参数直接保存在物品资产中，无需创建效果资产

**代码中使用**：

```csharp
// 触发效果（自动创建EffectContext并执行）
FishData fish = ...;
fish.TriggerRevealEffects();   // 揭示效果
fish.TriggerCaptureEffects();  // 捕获效果
fish.TriggerUseEffects();      // 使用效果

// 效果会自动查找玩家对象并执行
```

#### 3.4 当前支持的效果类型

| 效果类型 | 参数 | 说明 |
|---------|------|------|
| **增加体力** | numericValue | 增加固定数量的体力值 |
| **修改疯狂值** | numericValue | 修改世界疯狂值（正数增加，负数减少） |
| **随机增加体力** | minValue, maxValue | 在最小值和最大值之间随机增加体力 |

#### 3.6.1 新增效果类型注意事项

**参数类型选择**：
- 如果效果只需要一个固定数值，使用 `numericValue`
- 如果效果需要随机范围，使用 `minValue` 和 `maxValue`
- 如果效果需要多个不同类型的参数，需要扩展 `EffectConfig` 类添加新字段

**参数验证**：
- 在 `EffectConfigDrawer` 中添加参数验证逻辑
- 使用 `EditorGUI.HelpBox` 显示警告信息
- 例如：最小值不能小于0、最大值不能小于最小值等

**Inspector显示**：
- 确保参数字段有清晰的标签和工具提示
- 根据效果类型只显示相关参数，隐藏不需要的参数
- 保持布局一致性和可读性

**效果执行**：
- 从 `EffectContext` 中读取参数，不要从效果类内部字段读取
- 确保参数验证（如数值范围检查）
- 添加适当的日志输出便于调试
