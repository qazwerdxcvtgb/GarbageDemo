## 一、基础系统

基础系统包含游戏核心框架类，提供事件系统、状态管理、输入处理和交互机制。

### 1.1 GameEvents - 事件定义

**文件路径**：`Assets/Script/GameEvents.cs`

**功能说明**：定义项目中通用的事件类型和枚举

**核心内容**：

**IntValueChangedEvent**
- 整数值变化事件，继承自 `UnityEvent<int>`
- 用于传递整数类型参数（体力值、金币、疯狂值等）

**SanityLevel 枚举**
- 疯狂等级划分，共6个等级
- Level0：疯狂值为0
- Level1：疯狂值1-3
- Level2：疯狂值4-6
- Level3：疯狂值7-9
- Level4：疯狂值10-12
- Level5：疯狂值13+

**SanityLevelChangedEvent**
- 疯狂等级变化事件，继承自 `UnityEvent<SanityLevel>`

---

### 1.2 GameManager - 游戏管理器

**文件路径**：`Assets/Script/GameManager.cs`

**设计模式**：单例模式 + DontDestroyOnLoad

**功能说明**：管理全局事件系统和全局游戏状态

**全局事件**：

| 事件名 | 类型 | 说明 |
|--------|------|------|
| OnInteractionPressed | UnityEvent | 玩家按下交互键时触发 |
| OnSanityChanged | IntValueChangedEvent | 疯狂值变化时触发 |
| OnSanityLevelChanged | SanityLevelChangedEvent | 疯狂等级变化时触发 |

**核心属性**：

| 属性名 | 类型 | 说明 |
|--------|------|------|
| SanityValue | int | 当前疯狂值（≥0，无上限） |
| CurrentSanityLevel | SanityLevel | 当前疯狂等级（只读） |
| EmptyCardDrawnCount | int | 空牌抽取统计（只读） |

**核心方法**：

```csharp
// 交互事件
void TriggerInteractionPressed()

// 疯狂值管理
void ModifySanity(int amount)      // 修改疯狂值（正数增加，负数减少）
void SetSanity(int value)          // 直接设置疯狂值
int GetSanity()                    // 获取当前疯狂值
void ResetSanity()                 // 重置疯狂值为0

// 疯狂等级
SanityLevel CalculateSanityLevel(int sanity)  // 计算疯狂等级
SanityLevel GetSanityLevel()                   // 获取当前等级
string GetSanityLevelDescription(SanityLevel) // 获取等级描述

// 空牌统计
void RecordEmptyCardDrawn(int count)  // 记录空牌抽取
int GetEmptyCardDrawnCount()          // 获取统计
void ResetEmptyCardCount()            // 重置统计
```

**使用示例**：

```csharp
// 订阅疯狂值变化事件
GameManager.Instance.OnSanityChanged.AddListener(OnSanityChanged);

// 修改疯狂值
GameManager.Instance.ModifySanity(3);  // 增加3点疯狂值
GameManager.Instance.ModifySanity(-2); // 减少2点疯狂值

// 获取当前疯狂等级
SanityLevel level = GameManager.Instance.CurrentSanityLevel;
```

**命名空间**：`ItemSystem`

**目录结构**：

```

├── Data/                    # 数据定义
│   ├── ItemData.cs          # 物品基类

│   ├── EffectTrigger.cs     # 触发时机枚举
│   ├── EffectContext.cs     # 效果上下文
│   ├── InstantEffect.cs     # 即时效果基类
│   ├── PassiveEffect.cs     # 被动效果基类
```

**设计理念**：

**类继承关系**：

```

├── InstantEffect (即时效果基类)
│   ├── Effect_AddHealth
│   ├── Effect_AddRandomHealth
│   ├── Effect_ModifySanity
│   ├── Effect_DrawCards
│   └── Effect_RandomHealthOrSanity

```

### 2.3 效果层 - EffectBase多态系统

#### 2.3.1 核心机制

**技术实现**：
- 使用 `[SerializeReference]` 特性实现多态序列化
- 效果直接嵌入 ItemData 的 ScriptableObject 中
- 通过 EffectBaseDrawer 编辑器扩展提供 Inspector 下拉选择

**优势**：
- 无需为每个效果创建单独的 ScriptableObject 资产
- 效果参数直接在物品资产中配置
- 新增效果类型自动出现在下拉菜单中

---

#### 2.3.2 EffectBase（效果基类）

**文件路径**：`Assets/Script/ItemSystem/Effects/EffectBase.cs`

**类定义**：

```csharp
[System.Serializable]
public abstract class EffectBase
{
    [Tooltip("触发时机")]
    public EffectTrigger trigger = EffectTrigger.OnUse;
    
    public abstract string DisplayName { get; }
    public abstract void Execute(EffectContext context);
    public virtual string GetDescription() => DisplayName;
}
```

**关键点**：
- 标记为 `[System.Serializable]`（非ScriptableObject）
- `trigger` 字段指定效果触发时机
- `DisplayName` 用于 Inspector 显示
- `Execute()` 为效果执行逻辑

### 2.4 管理层

#### 2.4.1 ItemPool（物品池）

**文件路径**：`Assets/Script/ItemSystem/Managers/ItemPool.cs`

**设计模式**：单例模式 + DontDestroyOnLoad

**数据来源**：`Resources.LoadAll<ItemData>` 从以下路径加载：
- `Resources/Items/Fish/` - 鱼类
- `Resources/Items/Trash/` - 杂鱼
- `Resources/Items/Consumable/` - 消耗品
- `Resources/Items/Equipment/` - 装备

**分池结构（鱼类专用）**：

```
fragmentedFishPools: Dictionary<FishDepth, List<List<FishData>>>

每个深度(Depth1/Depth2/Depth3) 包含 3 个子池
├── 子池0: [鱼A, 鱼D, 鱼G, ...]
├── 子池1: [鱼B, 鱼E, 鱼H, ...]
└── 子池2: [鱼C, 鱼F, 鱼I, ...]

初始化时：
1. 按深度筛选所有鱼类
2. Fisher-Yates 洗牌
3. 轮询分发到 3 个子池
```

**核心方法**：

```csharp
// 鱼类抽取（关卡用）
List<FishData> GetCardsForStage(FishDepth currentStageDepth)
// 根据深度获取卡牌：Depth1返回3张，Depth2返回6张，Depth3返回9张
// 只读方法，不会移除卡牌

bool RemoveCard(FishData cardToRemove)
// 从分池中精准移除指定卡牌（用于捕获后）

// 鱼类筛选抽取（旧接口）
FishData DrawCardWithFilter(FishDepth? depth, FishSize? size)
List<FishData> DrawUniqueCards(int count, FishDepth? depth, FishSize? size)

// 通用抽取
ItemData DrawItem(ItemCategory category)
// 从指定类型牌堆抽取一张（会移除）

List<ItemData> DrawItemsByCategory(ItemCategory category, int count)
```

### 2.6 扩展指南

#### 2.6.1 添加新的物品类型

1. 创建新类继承 `ItemData`
2. 添加专属属性
3. 实现 `GetItemInfo()` 方法
4. 如需效果支持，添加 `[SerializeReference] List<EffectBase> effects` 字段

```csharp
public class NewItemData : ItemData
{
    public int newProperty;
    
    [SerializeReference]
    public List<EffectBase> effects = new List<EffectBase>();
    
    public override string GetItemInfo()
    {
        return $"{itemName}: {description}";
    }
}
```

#### 2.6.2 添加新的效果类型

1. 创建新类继承 `EffectBase`
2. 添加效果参数字段
3. 实现 `DisplayName` 属性
4. 实现 `Execute()` 方法
5. 可选：重写 `GetDescription()` 方法

```csharp
[System.Serializable]
public class Effect_NewEffect : EffectBase
{
    [Tooltip("效果参数")]
    public int parameter = 10;
    
    public override string DisplayName => "新效果";
    
    public override void Execute(EffectContext context)
    {
        // 效果逻辑
        Debug.Log($"执行新效果，参数: {parameter}");
    }
    
    public override string GetDescription()
    {
        return $"新效果描述 ({parameter})";
    }
}
```

**注意**：新效果类创建后会自动出现在 Inspector 的效果类型下拉菜单中，无需额外注册。

## 七、FishingSystem - 钓鱼系统

### 7.1 系统概述

**命名空间**：`FishingSystem`

**设计理念**：常驻式3×3牌堆钓鱼界面，玩家通过点击牌堆消耗体力揭示鱼类卡牌，选择捕获或放弃。

**目录结构**：

```
Assets/Script/FishingSystem/
├── FishingTablePanel.cs       # 钓鱼牌桌主控制器
├── CardPileSlot.cs            # 牌堆槽位控制器
└── RevealOverlayPanel.cs      # 揭示遮罩层控制器
```

**核心特性**：
- **9个牌堆槽位**：3行×3列，对应3个深度×3个子池
- **状态追踪**：每个牌堆追踪未翻开/已翻开/空状态
- **体力消耗机制**：揭示消耗1体力，捕获消耗卡牌对应体力
- **放弃机制**：放弃后卡牌正面朝上保留，获得杂鱼牌
- **与手牌交互**：遮罩显示期间手牌层可用

**依赖**：
- ItemSystem（ItemPool、FishData）
- FishCardSystem（FishCard、FishCardVisual）
- HandSystem（HandManager）
- CharacterState（体力管理）

**核心方法**：

**工作流程**：

```
1. Initialize() 初始化
   └─ 设置 depth 和 poolIndex
   └─ 调用 RefreshDisplay()

2. RefreshDisplay() 刷新显示
   └─ 调用 ItemPool.PeekTopCard(depth, poolIndex)
   └─ 如果有牌：实例化 FishCard，设置牌堆模式，显示卡背
   └─ 如果无牌：设置状态为 Empty

3. OnPointerClick() 点击
   └─ 触发 OnSlotClicked 事件
```

**启用牌堆模式**：


#### 7.6.5 常见问题

**Q1：牌堆槽位不显示卡牌？**
- 检查 ItemPool 是否正确初始化
- 检查 fishCardPrefab 是否分配
- 检查 cardSpawnPoint 是否正确设置

**Q2：点击牌堆没有反应？**
- 检查槽位是否有 Button 组件或实现了 IPointerClickHandler
- 检查 OnSlotClicked 事件是否正确订阅

**Q3：遮罩层挡住手牌？**
- 检查 Canvas 的 sortingOrder
- HandUIPanel 应该设置最高的 sortingOrder

**Q4：捕获后牌堆没有刷新？**
- 检查 ItemPool.RemoveCard() 是否成功执行
- 检查 slot.RefreshDisplay() 是否被调用
